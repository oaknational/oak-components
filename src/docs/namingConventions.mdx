# Naming Conventions and Exports

## Export Naming Conventions

- Any items prefixed with "Oak" are exported
- Any other items are not to be exported

## Component Naming Conventions

#### Oak[*ComponentName*]
- An Oak Component is any component which is exported in the library interface
- Oak components should not be nested in other component folders

#### Sub[*Parent*]ComponentName = a sub component of an exported component
- A Sub Components is any component that supports a single Oak Component
- Sub components should not be exported in the index file
- Sub components should be nested in their parent Oak Component's folder
- Sub components should not have their own stories

####  Internal[*ComponentName*] = 
- An Internal Component is a non exported component which supports several Oak Components
- Internal components should not be nested in other component folders
- Internal components should not be exported in the index file
- Internal components should have their own stories

## Style Name Conventions

### Parameters

#### oak[*ParameterName*] 
- an array of accepted parameter values

#### Oak[*ParameterName*]
- a union type of accepted parameter values

### Tokens

#### oak[*ParameterName*]Tokens
- an object of token names mapped to their corresponding values
- NB. values may be other tokens

#### Oak[*ParameterName*]Tokens
- A union type of token names

#### OakTheme
- A type describing the theme object

#### oak[*ThemeName*]Theme
- an exported theme object

#### [*KeyParameterName*]Map
- A type describing a map between the key parameter and corresponding tokens or values
- Record < Key, possible values >

#### [*ParameterName*]Style
- Css generated by styled components 
- Internal use only

#### [*ParameterName*]StyleProps
- The prop types for the parameter
- To be consumed by Oak Components
